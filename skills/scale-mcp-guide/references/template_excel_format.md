# テンプレート作成用 Excel フォーマット

ユーザーが Excel でテンプレート設計を提出した場合、この仕様に従って読み取り、`scale_create_template` の `axes` パラメータに変換する。

サンプルファイル: `templates/template_creation_sample.xlsx`

## 全体構成

Excel は評価制度設計用のワークブックで、以下のシートを参照する:

| 用途 | シート名 | 役割 |
|------|---------|------|
| **MCP 設定** | `テンプレート設定` | 基本情報 + 軸レベルの表示フラグ（必須） |
| **マスター特記事項** | `マスター設定` | マスター登録に関する特記事項（存在すれば最優先で参照） |
| 成果軸の項目 | `成果目標（KGI・KPI）` | ユーザー別 KGI/KPI・目標値・6段階尺度 |
| 成果項目マスタ | `成果項目事例一覧` | 成果項目候補リスト |
| KGI/KPI 定義 | `KGI`, `KPI` | 最低〜最高評価の定義 |
| バリュー軸の項目 | `バリュー評価４段階` | バリュー項目・評価基準（A/B/C/D） |
| バリューマスタ | `バリュー事例①CB`, `バリュー事例②コンピテンシー`, `コンピテンシーマスター`, `着眼点` | バリュー項目候補 |
| プロセス軸の項目 | `WOOP目標設定` | プロセス目標設定フォーマット |
| ユーザー情報 | `ユーザー情報` | 評価対象者リスト |
| シート分類 | `シート分類`, `評価体制` | 職種・職位・評価体制 |
| ウェイト設定 | `ウェイト` | シートごとのウェイト設計 |
| その他 | `基本給①②`, `賞与①②`, `評点シミュレーション` | 報酬制度（MCP対象外） |

MCP でテンプレートを作成する際は、`テンプレート設定` シートで軸の表示フラグを取得し、項目の中身は該当する項目マスタシートから読み取る。

## シート: マスター設定（存在する場合は最優先で確認）

マスター（評点表記・職位・職種・所属・ランク等）の登録方針に関する**特記事項**を格納するシート。Excel にこのシートがある場合、`テンプレート設定` を読む前にこの内容を必ず確認し、テンプレート作成フロー全体に反映する。

### カラム構成

| カラム | 内容 |
|-------|------|
| `No.` | 連番 |
| `マスター` | 対象マスター名（例: `評点表記`, `職位`, `職種`, `所属`, `ランク`） |
| `特記内容` | そのマスターを登録・選択する際の指示や解釈ルール |

### 解釈のルール

- 特記内容は他シートの**読み取り方針や MCP 引数の決め方をオーバーライド**する指示として扱う
- 特記対象のマスターがまだ Scale に存在しない場合は、特記内容に従って先に `scale_create_*` で登録してから `score_notation_id` 等を解決する
- 特記内容から生成したマスター値（例: 評点表記のラベル辞書）は、軸設定・項目設定と一緒に表形式でユーザーに提示し、承認を取る

### 例

サンプル Excel の `マスター設定` には以下のような特記がある:

> 評点表記 — 「バリュー評価４段階」シートにある「詳細」列に評点表記が書かれています。評価内容として例えば「A評価」とは、4点:A評価、3点:B評価、2点:C評価、1点:D評価 のように評点表記としてマスター登録してください。

この場合の処理:

1. `バリュー評価４段階` シートの「詳細」列から A/B/C/D 評価のラベル文言を抽出
2. `scale_create_score_notation` で `{"4": "A評価", "3": "B評価", "2": "C評価", "1": "D評価"}` のような 4 段階評点表記を登録（既に同名の評点表記があれば再利用）
3. バリュー軸の `score_notation_id` にそれを割り当てる
4. ユーザーに登録内容と軸への割り当てを提示して承認を得る

## シート: テンプレート設定（最重要）

### 基本情報セクション

キー・バリューの2カラム形式:

| 項目 | 値の例 |
|------|-------|
| テンプレート名 | 営業_部長級_営業部 |
| 説明 | 営業部長級向けの評価テンプレート |
| 評価期間 | 2026年7月～2026年12月 |
| 総合コメント表示 | あり / なし |
| 評価段階数 | 2 / 3 |

**MCP マッピング:**
- テンプレート名 → `name`
- 説明 → `description`
- 評価期間 → （`evaluation_period_id` に変換、`scale_list_evaluation_periods` で照合）
- 総合コメント表示 → `show_overall_comment` (`あり`→`true`, `なし`→`false`)
- 評価段階数 → `evaluation_steps`

### 評価軸設定セクション

1行目がヘッダ、2行目以降が各軸のデータ:

| 軸名 | ウェイト | 目標表示 | 軸尺度 | 詳細表示 | 中間評価 | 自己評点 | 自己コメント | 評価者評点 | 評価者コメント | 評点表記 | 目標タイプ |
|-----|--------|---------|-------|---------|---------|---------|-----------|----------|-------------|---------|----------|
| 成果 | 70 | ○ | × | ○ | ○ | ○ | ○ | ○ | × | 6段階評価 | outcome |
| バリュー | 20 | × | × | ○ | ○ | ○ | ○ | ○ | ○ | 4段階評価 | behavior |
| プロセス | 10 | ○ | × | × | ○ | ○ | ○ | ○ | ○ | 4段階評価 | other |

**記号の意味:**
- `○` = 有効 / 表示する
- `×` = 無効 / 表示しない
- 空白 = デフォルト値を使用

**MCP マッピング（軸ごと）:**

| Excel カラム | MCP パラメータ |
|------------|--------------|
| 軸名 | `name` |
| ウェイト | `weight`（0-100の整数、全軸合計100） |
| 目標表示 | `show_goal` |
| 軸尺度 | `show_scale` |
| 詳細表示 | `show_detail` |
| 中間評価 | `has_interim_evaluation` |
| 自己評点 | `final_has_self_score` / `interim_has_self_score` |
| 自己コメント | `final_has_self_comment` / `interim_has_self_comment` |
| 評価者評点 | `final_has_evaluator_score` |
| 評価者コメント | `final_has_evaluator_comment` / `interim_has_evaluator_comment` |
| 評点表記 | `score_notation_id`（マスタ名から解決） |
| 目標タイプ | `objective_type` (`outcome` / `behavior` / `other`) |

## 項目データの取得元

`テンプレート設定` には評価項目の詳細は含めない。項目は以下のシートから取得する:

### 成果軸の項目

**優先順: `成果目標（KGI・KPI）` > `成果項目事例一覧`**

`成果目標（KGI・KPI）` シートのカラム構成:
- ユーザー基本情報（No, 社員番号, 氏名, 職種, 職位, 所属）
- **成果目標1（50）**: 項目名
- 目標値、1点、2点、3点、4点（目標）、5点、6点 — 項目尺度ラベルに対応
- **成果目標2（25）**: 2つ目の項目（同形式）
- **成果目標3（25）**: 3つ目の項目（同形式）

括弧内の数値（50, 25, 25）は項目ウェイト。各ユーザーごとに3つの成果目標を持つ設計。

**MCP マッピング（成果軸の項目）:**
- 項目名 → `name`
- 目標値 → `goal`
- ヘッダの括弧内数値 → `weight`
- 1点〜6点 → `item_scale_labels` = `{"1": "1点の値", "2": "2点の値", ..., "6": "6点の値"}`
- `show_item_scale` は自動的に `true` となる

### バリュー軸の項目

**シート: `バリュー評価４段階`**

各行が1項目。列の例:
- A1: 項目名（例: 明るさ・誠実さ）
- A2: 期待される行動 + A/B/C/D評価基準（改行区切り、詳細テキスト）

**MCP マッピング（バリュー軸の項目）:**
- 項目名 → `name`
- 評価基準全文 → `detail`
- `goal` は空（`show_goal=false`）
- `item_scale_labels` は空（`show_item_scale=false`）
- `weight` は均等（例: 全項目 1）

### プロセス軸の項目

**シート: `WOOP目標設定`**

WOOP（Wish / Outcome / Obstacle / Plan）形式の目標設定。

- 各行がプロセス目標
- 目標内容を `goal` または `detail` にマッピング
- 詳細な評価基準は `detail` に含める

## 読み取り処理の流れ

1. **`マスター設定` シートを確認**（存在する場合）
   - 特記内容を抽出し、後続ステップの読み取り方針・マスター登録に反映する
   - 新規マスター登録が必要なら先に `scale_create_*` を実行して ID を解決
2. **`テンプレート設定` シートを読み取る**
   - 基本情報セクションから `name`, `description`, `evaluation_period_id` 等を取得
   - 評価軸設定セクションから軸のフラグ（`show_goal`, `final_has_evaluator_comment` 等）を取得
3. **評価期間名から `evaluation_period_id` を解決** （`scale_list_evaluation_periods`）
4. **評点表記名から `score_notation_id` を解決** （`scale_list_score_notations`、マスター設定の特記で新規作成したものも含む）
5. **軸ごとに項目を各マスタシートから取得**
   - 成果軸 → `成果目標（KGI・KPI）`（ユーザー別）または `成果項目事例一覧`（マスタ）
   - バリュー軸 → `バリュー評価４段階`
   - プロセス軸 → `WOOP目標設定`
6. **変換結果をユーザーに表形式で提示して確認**
7. **承認後、`scale_create_template` を実行**

## 読み取り時の検証

Excel を読み取った後、以下を検証してから `scale_create_template` を実行する:

1. **軸ウェイト合計** が 100 であること
2. **軸数** が 5 以下であること
3. **軸ごとの項目数** が 100 以下であること
4. **評点表記名** が `scale_list_score_notations` の結果に存在すること
5. **評価期間名** が `scale_list_evaluation_periods` の結果に存在すること

検証エラーがある場合は、ユーザーに確認して修正を促す。

## ユーザー別 vs 共通テンプレート

`成果目標（KGI・KPI）` シートはユーザー別に目標を持つ。以下のいずれかで対応:

**パターンA: 職種・職位別にテンプレートを作成**
- 同じ職種・職位のユーザーは同じ項目を使う（ユーザー1人分の行を代表として読み取る）
- `scale_distribute_sheets` で配布時にユーザーを選ぶ

**パターンB: 個別シートとして配布**
- ユーザーごとに別テンプレートを作成（現実的でないことが多い）

MCP では**パターンA** を推奨。ユーザーに「どの行を代表として使うか」を確認する。

## 例: 添付 Excel の変換結果

### 基本情報
- `name`: "営業_部長級_営業部"
- `evaluation_period_id`: （"2026年7月～2026年12月"から解決）
- `show_overall_comment`: true

### 評価軸

| 軸 | weight | show_goal | show_scale | show_detail | show_item_scale | has_interim_evaluation | final_has_evaluator_comment | objective_type |
|----|--------|-----------|-----------|-----------|----------------|----------------------|---------------------------|--------------|
| 成果 | 70 | true | false | true | true | true | **false** | outcome |
| バリュー | 20 | **false** | false | true | false | true | true | behavior |
| プロセス | 10 | true | false | **false** | false | true | true | other |

### 項目の取得元

- 成果軸の3項目 → `成果目標（KGI・KPI）` の特定ユーザー（例: 山田 太郎）の行から読み取る
- バリュー軸の項目 → `バリュー評価４段階` の全項目
- プロセス軸の項目 → `WOOP目標設定` の該当行
